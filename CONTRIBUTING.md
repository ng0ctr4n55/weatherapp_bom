# Contributing to WeatherApp

We welcome contributions from the community! Whether you are fixing bugs, improving documentation, or adding new features, your help is greatly appreciated.

Please read this guide before submitting a pull request.

## 🚀 Getting Started

1.  **Fork the Repository:** Fork this repository to your GitHub account.
2.  **Clone Locally:** Clone your fork to your local machine:
    ```bash
    git clone <YOUR_FORK_URL>
    cd weatherapp
    ```
3.  **Create a Branch:** Create a new branch for your changes. It's best practice to name your branch descriptively (e.g., `feat/add-hourly-forecast` or `fix/api-key-error`).
    ```bash
    git checkout -b your-feature-branch-name
    ```
4.  **Make Changes:** Implement your changes and write any necessary tests.
5.  **Commit:** Commit your changes with clear, descriptive messages.
    ```bash
    git add .
    git commit -m "feat: A concise summary of the change made"
    ```
6.  **Push & Pull Request:** Push your branch to your fork and open a Pull Request (PR) against the `main` branch of the original repository.

## ✨ Feature Development

When adding a new feature:
*   **Design First:** Think about the user experience and the necessary API endpoints/data models.
*   **Write Tests:** Write unit and integration tests *before* or *while* writing the feature code.
*   **Keep it Small:** Break large features into smaller, manageable PRs.

## 🐛 Bug Fixes

1.  **Reproduce:** Ensure you can reliably reproduce the bug on your local machine.
2.  **Isolate:** Write a minimal test case that demonstrates the bug. This test should fail before your fix and pass after your fix.
3.  **Fix:** Apply the necessary code changes.

## 📝 Documentation

If you add a new feature or change a significant part of the codebase, please update the relevant documentation files (e.g., `README.md`, API documentation).

## 🤝 Code Review

All contributions are welcome, but please be prepared to discuss your changes. We value constructive criticism, and we ask that you be open to feedback on your PRs.
